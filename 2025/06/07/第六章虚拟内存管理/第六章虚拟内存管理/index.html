第六章虚拟内存管理

发表于2025-06-07|更新于2025-06-07|操作系统
|浏览量:

1. 虚拟存储器的基本概念

虚拟存储器是操作系统用来扩展物理内存的一种技术，它使得程序感觉它拥有一个连续的内存空间，虽然实际内存可能分布在物理内存和磁盘之间。通过将物理内存和磁盘结合使用，虚拟存储器让程序可以运行在比实际物理内存更大的地址空间中。

  • 逻辑地址：程序在运行过程中生成的地址，也称为虚拟地址。程序员通常使用的是逻辑地址，这些地址会在操作系统的帮助下转换为物理内存中的真实地址。
  • 线性地址：逻辑地址经过映射后的中间地址。在某些系统中，线性地址是逻辑地址和物理地址之间的一个过渡地址。
  • 物理地址：实际存在于内存中的地址。它是物理内存的真实地址，在计算机硬件中可以直接访问。

虚拟存储器的目标是让程序在执行时不再受物理内存大小的限制。通过在内存和磁盘之间调度数据，虚拟存储器能够让程序“觉得”自己有一个比实际物理内存更大的内存空间。

2. 虚拟存储器的优点

  • 地址空间大于物理内存：

    • 程序大小不再受限：使用虚拟存储器，程序不再需要全部装入物理内存。用户可以编写比物理内存大得多的程序，虚拟存储器将程序的某些部分（例如不活跃的部分）存储在磁盘上，按需调入内存。
    • 内存扩展：程序可以访问一个大于物理内存的地址空间，不同程序可以共享相同的物理内存区域，从而提高内存的利用率。

  • 提高内存利用率：

    • 按需加载：只将程序中需要执行的部分加载到内存中，其它部分仍然保留在磁盘上，直到程序需要时才加载。这减少了内存的浪费，特别是在运行多个进程时，内存得到了更充分的利用。
    • 更高效的内存分配：通过动态加载与替换，操作系统能够保证内存资源得到最大化的利用。

  • 支持多个进程并发执行：

    • 虚拟存储器使得多个进程可以并发执行，即使它们总共占用的内存量超过了物理内存的大小。操作系统通过将部分进程的内容交换到磁盘来管理内存空间。
    • 通过虚拟存储器，系统能够容纳更多的进程并行执行，从而提高系统的吞吐量和效率。

3. 虚拟存储器的引入

在传统的存储管理方案中，程序必须完全加载到内存中才能运行，这存在以下几个问题：

  • 程序过大：当程序的大小超过物理内存时，无法完全加载到内存中，导致程序无法运行。
  • 资源利用率低：由于只能运行少量的程序，计算机的资源（如内存和CPU）不能得到充分利用。程序之间无法共享内存，可能导致空闲内存的浪费。

虚拟存储器的解决方法：

  • 按需加载：虚拟存储器的引入通过将部分程序存放在磁盘上，按需调入内存来解决程序过大的问题。
  • 更高效的内存管理：虚拟存储器不仅让程序超越物理内存的限制，还允许操作系统将多个进程的执行内容存储在内存和磁盘中，减少了内存空闲的浪费，提高了资源利用率。

4. 局部性原理（Locality of Reference）

局部性原理是虚拟存储器的核心概念之一。程序在执行过程中，通常会频繁地访问某些内存区域。这些内存区域的访问呈现出一定的规律，通常包括：

  • 时间局部性：某条指令在某时刻执行后，不久后会再次执行。例如，循环中的变量或方法会被重复访问。

    • 示例：一个程序循环遍历数组，某些数据将被频繁访问。

  • 空间局部性：程序访问某个存储单元后，可能会访问附近的存储单元。例如，访问一个数组元素后，程序通常会访问数组中的下一个元素。

    • 示例：在遍历数组时，访问的元素通常是相邻的，因此会引发对相邻内存地址的访问。

虚拟存储器通过局部性原理来优化内存使用。操作系统利用程序的局部性特征，将最常用的页面或数据优先加载到内存中，减少了磁盘访问的频率，提高了系统的性能。

总结

  • 虚拟存储器使得程序可以在一个比物理内存更大的地址空间中运行，同时通过内存和磁盘的协同工作，优化了内存利用率。

  • 优点：

    • 提供比物理内存更大的地址空间；
    • 通过按需加载提高内存利用率；
    • 支持多个进程并发执行，提升系统效率。

  • 虚拟存储器的引入解决了程序无法完全加载到内存中的问题，提高了系统的资源利用率。

  • 局部性原理：程序在执行时具有时间局部性和空间局部性，通过这两个特征，操作系统能够更高效地管理内存，提高系统性能。

这些概念是虚拟存储器管理的基础，对于理解现代操作系统如何有效地进行内存管理至关重要。如果你对某个部分有进一步的疑问，随时可以继续提问！

5. 虚拟存储器的功能

虚拟存储器提供了多种功能，使得程序能够运行在一个比物理内存更大的虚拟地址空间中。通过合理地管理内存，操作系统优化了资源的使用效率。

请求调入功能

  • 功能描述：当程序执行时，不是所有的页面都需要立即加载到内存中。虚拟存储器通过请求调入功能，根据程序的需求将未被访问的页面从磁盘调入内存，确保系统不浪费内存空间。

  • 具体操作：

    • 当程序需要访问某个页面时，如果该页面未在内存中，则操作系统会触发缺页中断，并从磁盘调入该页面。
    • 在此过程中，未被访问的页面可以留在磁盘上，而不需要占用宝贵的内存空间。

置换功能

  • 功能描述：当内存已满且需要调入新页面时，操作系统通过置换算法将内存中的部分页面调出，腾出空间给新页面。这是虚拟存储器管理中的重要功能。

  • 具体操作：

    • 系统使用页面置换算法（如FIFO、LRU等）来选择当前内存中最不常用或最久未使用的页面进行置换。
    • 置换出的页面会被保存到磁盘上，供下次需要时调入内存。
    • 置换功能确保系统能高效地管理有限的内存资源，避免因为内存不够用而导致系统崩溃。

6. 如何实现虚拟内存

虚拟内存使得程序可以访问比物理内存更大的地址空间，通过动态调入和置换内存页面来实现。

装入部分内存页面

  • 内存页装载：并非程序的所有页面都被访问到，操作系统只会将程序中需要的部分页面加载到内存中，而其他未被访问的页面则留在磁盘中。
  • 优化内存利用：这种按需装载的方式避免了程序不必要的内存占用，从而有效提升了内存的利用率。

请求调页（Demand Paging）

  • 工作原理：请求调页是虚拟内存管理中的一种核心技术。当程序访问一个尚未加载到内存中的页面时，操作系统会触发一个缺页中断，即页面缺失中断。

  • 处理过程：

    • 操作系统根据缺页中断的请求，从磁盘加载相应的页面到内存中，并更新页表。
    • 此时，程序可以继续运行，直到下一次需要访问尚未加载的页面。

请求调页的性能分析

  • 有效访问时间（Effective Access Time, EAT）：是访问内存的平均时间，考虑了缺页中断的发生和页面调入的时间。
  • 缺页率：是指程序访问的页面中，未加载到内存中的页面的比例。缺页率越高，意味着程序频繁进行页面调入操作，导致性能下降。

性能分析：

  • 通过计算有效访问时间，操作系统可以评估虚拟存储器管理的效率。如果缺页率过高，操作系统可能需要调整页面置换策略，或者增加内存容量以提高性能。

7. 页面置换算法（Page Replacement Algorithms）

当内存中的所有页框都已被分配且需要调入新页面时，操作系统通过页面置换算法来决定哪些页面需要被淘汰。这些算法的目的是选择最合适的页面进行置换，以保证系统的高效运行。

FIFO（First In First Out）

  • 原理：FIFO算法选择最早进入内存的页面进行淘汰，简单实现。
  • 优点：易于实现。
  • 缺点：不一定最优，可能会淘汰那些即将被频繁访问的页面，导致Belady异常（FIFO可能引起内存使用效率下降）。

OPT（Optimal）

  • 原理：OPT算法选择将来最久未使用的页面进行淘汰。理想情况下，OPT能够最小化缺页率。
  • 优点：理论上是最优的页面置换算法，能够实现最少的缺页中断。
  • 缺点：实现起来不实际，因为无法预测未来的页面访问模式。

LRU（Least Recently Used）

  • 原理：LRU算法选择最久未被使用的页面进行淘汰。它近似于OPT，试图通过访问历史来预测未来的访问模式。
  • 优点：性能较好，能够较为准确地反映页面的使用频率。
  • 缺点：需要额外的时间和空间来跟踪页面的访问历史，增加了系统开销。

Clock算法（Second Chance Replacement）

  • 原理：Clock算法是LRU的一种近似实现。它通过一个圆形队列来管理页面，每个页面都有一个访问位。访问位为0的页面被视为候选页面进行淘汰，而访问位为1的页面会获得“第二次机会”。
  • 优点：比LRU更高效，因为它只需一位位标记，而不需要追踪访问历史。
  • 缺点：虽然是LRU的近似算法，但并不完美，可能会产生更多的缺页中断。

8. 请求调页的具体实现

请求调页过程

  1. 缺页中断：当程序访问一个尚未加载到内存中的页面时，会触发一个缺页中断。这种中断将停止程序的执行，转交给操作系统处理。

  2. 页面加载：

    • 操作系统通过调入机制，从磁盘加载所需的页面到内存中。这通常涉及将页面从磁盘的交换空间或文件系统加载到内存的空闲页框中。
    • 操作系统根据页面置换算法来选择哪些页面需要被淘汰，腾出空间给新页面。

  3. 更新页表：一旦页面被加载到内存中，操作系统会更新该进程的页表，标记该页面现在在内存中。

  4. 恢复执行：缺页处理完成后，程序恢复执行，访问刚刚加载的页面。

总结：

  • 请求调页是虚拟存储器管理的核心功能之一，通过按需加载页面，减少内存的浪费。
  • 操作系统使用页面置换算法来决定哪些页面需要被淘汰，确保内存能够容纳当前执行所需的页面。
  • 虽然页面置换会引入额外的性能开销，但通过优化算法和减少缺页率，可以有效提高系统性能。

通过对请求调页、页面置换算法及其实现的详细理解，能够更好地掌握虚拟存储器的工作机制，并能优化操作系统的内存管理。如果您对某个部分有进一步的问题，欢迎继续提问！

9. 页面淘汰策略 - 示例

页面淘汰策略用于确定当内存已满，且需要调入新页面时，哪个页面应该被淘汰以腾出空间。这些策略的选择直接影响系统的性能，下面我会分别通过实际例子来讲解FIFO、LRU、Clock算法等常见的页面置换策略。

1. FIFO（First In First Out）页面置换

原理：FIFO是最简单的页面置换策略。操作系统维护一个队列，队列中的页面按照进入内存的顺序排列。当内存满时，选择队列中最早进入内存的页面进行淘汰。

示例：
假设我们有 4 个页面框架（即内存容量为 4 页），并且程序访问的页面序列为：A, B, C, D, A, E, B, C, D, A。

  • 初始时，内存为空，依次加载页面A、B、C、D，内存状态为：[A, B, C, D]。
  • 当访问页面A时，页面A已经在内存中，因此没有发生缺页中断。
  • 当访问页面E时，由于内存已满，FIFO算法淘汰队列中最早加载的页面（即页面A），并将页面E加载到内存中，内存状态变为：[B, C, D, E]。
  • 继续访问页面B，页面B已经在内存中，不发生缺页中断。
  • 访问页面C，页面C已在内存中，不发生缺页中断。
  • 访问页面D，页面D已在内存中，不发生缺页中断。
  • 最后访问页面A时，页面A需要被调入内存，而FIFO会淘汰队列中最早进入内存的页面（即页面B），内存状态变为：[A, C, D, E]。

总结：FIFO的最大问题是，它可能淘汰那些将来会频繁访问的页面，导致不必要的页面调入调出，从而降低系统性能。

2. LRU（Least Recently Used）页面置换

原理：LRU算法选择最久未使用的页面进行淘汰。通过追踪每个页面的最近使用时间，当需要淘汰页面时，选择最后一次使用时间最久的页面。

示例：
假设我们有 4 个页面框架，页面访问序列为：A, B, C, D, A, E, B, C, D, A。

  • 初始时，内存为空，依次加载页面A、B、C、D，内存状态为：[A, B, C, D]。
  • 当访问页面A时，页面A已经在内存中，更新它的访问时间。
  • 当访问页面E时，LRU会选择淘汰最久未被使用的页面（即页面B），并将页面E加载到内存中，内存状态变为：[A, C, D, E]。
  • 继续访问页面B，LRU会再次淘汰最久未被使用的页面（即页面C），内存状态变为：[A, B, D, E]。
  • 访问页面C时，LRU会淘汰最久未被使用的页面（即页面D），内存状态变为：[A, B, C, E]。
  • 最后访问页面A时，页面A已经在内存中，不发生缺页中断。

总结：LRU在理论上较为优秀，因为它考虑了程序的时间局部性，频繁访问的页面将不会被淘汰。然而，LRU的缺点在于需要追踪页面的访问历史，因此会增加一定的管理开销。

3. Clock算法（Second Chance Replacement）

原理：Clock算法是一种基于LRU的近似算法。它使用一个指针（像时钟一样循环）指向页面队列中的页面，并为每个页面设置一个访问位（bit）。当需要淘汰页面时，指针会扫描页面，如果页面的访问位是0，则该页面被淘汰；如果访问位是1，则给该页面一个“第二次机会”，访问位被清零，指针继续向前扫描。

示例：
假设我们有 4 个页面框架，页面访问序列为：A, B, C, D, A, E, B, C, D, A，且页面的初始访问位为0。

  • 初始时，内存为空，依次加载页面A、B、C、D，内存状态为：[A, B, C, D]，访问位为：[0, 0, 0, 0]。
  • 当访问页面A时，页面A已经在内存中，更新页面A的访问位为1，内存状态不变。
  • 当访问页面E时，由于内存已满，指针扫描并找到页面A，访问位为1，给页面A第二次机会，清零其访问位，继续扫描指针，找到页面B，访问位为1，给页面B第二次机会，继续扫描，找到页面C，访问位为0，淘汰页面C，加载页面E，内存状态为：[A, B, D, E]，访问位为：[1, 1, 0, 1]。
  • 继续访问页面B，页面B已在内存中，不发生缺页中断。
  • 访问页面C时，淘汰页面D，加载页面C，内存状态变为：[A, B, C, E]。
  • 最后访问页面A时，页面A已经在内存中，不发生缺页中断。

总结：Clock算法通过“第二次机会”机制避免了频繁的页面置换，能够在一定程度上近似LRU，同时降低了管理开销。

总结：

  1. FIFO：最简单的页面置换算法，容易实现，但不总是最优。它可能导致Belady异常，导致缺页率增加。
  2. LRU：选择最久未使用的页面进行淘汰，较为理想但需要额外的存储和管理开销。
  3. Clock算法：LRU的近似实现，通过给页面提供第二次机会，避免频繁的地址更新，管理开销较低，但可能无法完美模拟LRU的行为。

10. 工作集模型

工作集模型是用于优化内存管理的一种方法，旨在根据程序的局部性来动态分配内存，从而减少缺页中断。

工作集的定义

  • 工作集（Working Set）：指的是进程在某一时间窗口内访问的不同页面集合。即进程在一定时间范围内最常访问的页面，这些页面构成了进程的工作集。

工作集大小（Working Set Size）

  • 工作集大小：进程在一定时间窗口内所需的页面数量。它动态地反映了进程的内存需求。
  • 操作系统根据工作集大小来分配内存，确保进程能够在内存中拥有足够的页面，以避免频繁的缺页中断。

动态调整进程页面分配

  • 通过动态调整分配给每个进程的内存页面数量，操作系统能够优化内存的使用，防止过多的页面置换和缺页中断。

避免抖动（Thrashing）

  • 抖动（Thrashing）：指的是系统频繁进行页面置换，导致大量的缺页中断和低效的内存使用，严重影响系统性能。
  • 通过工作集模型，操作系统可以监控进程的内存需求和缺页率，调整每个进程的页面分配，防止抖动现象的发生。

11. 缺页率与帧分配

缺页率（Page Fault Rate）

  • 定义：缺页率是指程序访问页面时，所请求的页面不在内存中，需要从磁盘调入的频率。它通常用百分比表示。

  • 影响：缺页率越高，表示程序频繁访问未加载到内存中的页面，导致频繁的磁盘I/O操作，进而影响系统性能。

  • 计算：

    缺页率=缺页次数总访问次数\text{缺页率} = \frac{\text{缺页次数}}{\text{总访问次数}} 缺页率=总访问次数缺页次数​

帧分配（Frame Allocation）

  • 定义：帧分配是指操作系统将物理内存分配给每个进程的内存页框（page frames）。帧分配的目的是保证每个进程能有足够的内存页面，而又不浪费过多内存资源。

  • 过少的帧数：如果为进程分配的内存帧数过少，则进程可能无法容纳足够的页面，导致缺页率升高，频繁的页面调入会导致性能下降。

  • 过多的帧数：如果为进程分配的内存帧数过多，则会浪费内存空间，导致内存利用率下降。

分配策略

  • 分析工作集和缺页率：操作系统可以通过分析进程的工作集和缺页率来动态调整进程的内存帧分配。

    • 当缺页率较高时，操作系统可能会增加分配给进程的内存帧数，以减少缺页中断。
    • 当缺页率较低时，操作系统可能会减少内存帧数，避免过多占用物理内存。

减少抖动（Thrashing）

  • 通过合理的帧分配和工作集管理，操作系统可以避免系统因频繁的页面置换而产生的抖动现象，保持系统的高效运行。

总结

  • 页面淘汰策略：如FIFO、LRU、Clock算法等，帮助操作系统管理内存页面，确保系统的高效运行。每种策略有其优缺点，适用于不同的内存管理场景。
  • 工作集模型：通过计算并动态调整进程的工作集大小，优化内存分配，减少缺页中断，并防止抖动现象。
  • 缺页率与帧分配：合理的缺页率控制和帧分配能够提升系统性能，减少频繁的磁盘I/O操作，提高内存的利用效率。
