第5章 存储管理

1. 内存的基本目的

  • 内存的基本目的：内存是计算机的核心组件之一，用于存储正在执行的程序和数据。在计算机中，程序的执行是计算机完成任务的基本任务，而内存管理的主要任务是合理地分配内存资源，使得多个程序能够共享有限的内存资源。内存是程序执行的基础，它支持程序存储指令和数据，保证计算机的正常运行。

  • 内存管理的目标：为了提高内存的利用率和执行效率，操作系统需要进行合理的内存管理。操作系统通过不同的分配策略来控制多个进程对内存资源的访问和分配。

    • 合理分配内存：内存资源有限，因此操作系统需要合理分配内存空间，确保多个进程能够高效地共享这些资源。
    • 保护进程间的内存空间：内存管理还需要避免进程间的内存冲突，保护每个进程的内存空间不被其他进程非法访问。
    • 提高系统性能：通过有效的内存管理，操作系统能够提高系统性能，降低因内存碎片等问题带来的资源浪费。

2. 程序的执行和内存管理

程序执行的流程通常分为几个阶段：编译、链接、加载和重定位。

  1. 编译：

    • 过程：将源代码转换为汇编代码或中间代码。编译器检查程序的语法并生成汇编代码。
    • 作用：为程序的后续执行做准备，确保程序能够被正确转换为计算机可以理解的机器指令。

  2. 链接：

    • 过程：将编译生成的汇编代码与库文件（如标准库、外部库等）进行链接，生成可执行文件。
    • 作用：将程序的所有必要组件和依赖文件链接在一起，使得程序可以独立运行。
    • 静态链接：所有必要的库文件都被直接包含在可执行文件中，程序运行时无需再查找外部库。
    • 动态链接：程序在运行时加载所需的共享库，节省磁盘空间。

  3. 加载：

    • 过程：将生成的可执行文件加载到内存中，为程序执行做好准备。
    • 作用：将程序的指令和数据从硬盘传输到内存，以便CPU能够访问和执行这些指令。
    • 内存分配：加载时，操作系统为进程分配适当的内存区域，并将程序的代码段、数据段、堆和栈等部分装载到内存中。

  4. 重定位：

    • 过程：程序的内存地址在运行时可以动态修改。操作系统将可执行文件中的地址进行修改，使得程序能够在不同的内存位置运行。
    • 作用：通过动态地址重定位，操作系统能够确保程序在任何可用的内存位置上执行，而不必依赖固定的内存地址。
    • 静态重定位：程序的内存地址在编译时就已经固定，无法改变。
    • 动态重定位：操作系统可以在程序加载时修改程序的地址，使得程序能够在不同的内存位置运行。

3. 内存管理的基本概念

内存管理的基本概念是如何有效利用计算机的内存资源，并确保多个进程之间能够安全地共享这些资源。主要的内存分配方法包括连续内存分配和不连续内存分配。

连续内存分配：

连续内存分配是指操作系统将所有内存空间分成若干个区域，每个进程在某个连续的内存区域内运行。这种方式下，每个进程的内存位置是连续的。

  1. 等长分区：

    • 定义：将内存划分为若干个大小相同的分区，每个分区可以分配给一个进程。每个进程请求的内存大小固定，与分区大小相同。
    • 优点：实现简单，管理容易。
    • 缺点：如果进程的内存需求较小，分配给它的内存会浪费；如果进程的内存需求较大，可能会没有足够的空间。

  2. 不等长分区：

    • 定义：根据每个进程的实际内存需求，动态地划分内存分区。每个进程请求的内存大小不固定，可以根据需求分配合适大小的内存。
    • 优点：内存分配更为灵活，能有效避免内存浪费。
    • 缺点：可能导致内存碎片，管理较为复杂。
内存分配算法：

内存分配过程中，操作系统需要根据进程的内存需求和系统的空闲内存来做出合理的资源分配决策。常见的内存分配算法包括：

  1. 最佳适应算法（Best Fit）：

    • 原理：选择一个最小的空闲内存块，以便恰好满足进程的内存需求。目的是尽量减少剩余的空闲空间。
    • 优点：能够最大限度地减少内存碎片。
    • 缺点：可能导致过多的小碎片。

  2. 最先适应算法（First Fit）：

    • 原理：从头开始扫描空闲内存块，选择第一个足够大的空闲内存块来分配给进程。
    • 优点：实现简单，效率较高。
    • 缺点：可能导致内存的浪费和外部碎片。

  3. 循环适应算法（Next Fit）：

    • 原理：从上次分配的内存块位置开始搜索下一个足够大的内存块，直到找到适合的块。
    • 优点：避免了频繁从头开始扫描内存的开销。
    • 缺点：可能导致较大的碎片。

  4. 最差适应算法（Worst Fit）：

    • 原理：选择最大的空闲内存块来分配，以期产生较大的剩余空间，防止产生过小的碎片。
    • 优点：减少了分配较小内存块的可能。
    • 缺点：可能导致内存浪费，并且大块的内存块分配容易造成不均衡。

总结

  • 内存的基本目的是为了支持程序的执行，内存管理需要合理分配有限的内存资源，并确保多个进程能够共享这些资源而不发生冲突。
  • 程序执行的流程包括编译、链接、加载和重定位，这些步骤帮助程序顺利执行并确保内存地址的有效管理。
  • 内存管理的基本概念包括连续内存分配和不连续内存分配，分别通过等长分区和不等长分区来管理内存。
  • 内存分配算法的设计需要平衡内存的利用率和碎片问题，常见的算法有最佳适应、最先适应、循环适应和最差适应等。

这些内容为内存管理的基本框架，有助于理解操作系统如何有效管理内存并支持程序的执行。如果您有其他问题或需要更详细的解释，请随时告诉我！

4. 内存分配方案

内存分配方案是操作系统为管理计算机内存而采用的不同策略。根据分配方式的不同，内存管理方案可以分为固定分区和可变分区两种类型。

固定分区（Fixed Partitioning）

  • 原理：将整个内存划分为多个固定大小的分区，每个分区可以分配给一个进程。进程的内存需求必须小于或等于分区的大小。如果一个进程的实际内存需求小于分配的内存块大小，剩余部分会被浪费。

  • 优点：

    • 实现简单，管理起来比较容易。
    • 进程的内存分配和回收比较简单，不需要复杂的计算。

  • 缺点：

    • 内存浪费：如果进程所需的内存小于分配的内存块，就会造成内存的浪费（即内部碎片）。
    • 灵活性差：固定分区方式不适应不同大小的进程需求，可能导致某些进程无法获得足够的内存，或者内存分配不均。
可变分区（Variable Partitioning）

  • 原理：根据进程的实际内存需求，操作系统动态地将内存划分为多个大小不等的区域，每个进程请求的内存大小恰好等于其实际需求。

  • 优点：

    • 更灵活：根据进程的实际需求分配内存，不会浪费内存。
    • 内存利用率高：相比于固定分区，可变分区能够充分利用内存。

  • 缺点：

    • 内存碎片：可变分区可能会导致外部碎片问题，即内存中可能会出现不连续的小空闲区域，无法满足某些进程的内存请求。

5. 内存碎片

内存碎片是指在内存分配和回收过程中，出现了无法有效利用的空闲内存区域。内存碎片的产生通常是由于动态分配内存和进程释放内存的过程。

内部碎片（Internal Fragmentation）

  • 定义：分配给进程的内存中，未被使用的部分称为内部碎片。由于内存分配时按照固定大小分区，可能会分配一个大于进程实际需求的内存块，导致浪费。

  • 示例：假设进程A需要7KB的内存，而分配的内存块为8KB，那么剩余的1KB就形成了内部碎片。

外部碎片（External Fragmentation）

  • 定义：外部碎片是指系统中有多个小的空闲内存区域，但这些区域的大小不足以满足新进程的请求。虽然系统中总体上有足够的空闲内存，但这些内存块分散，不连续，无法被有效利用。

  • 示例：假设系统有10KB、5KB和3KB的空闲内存块，进程请求8KB内存，系统无法满足，因为没有一个单独的空闲块能满足请求，虽然总内存是15KB。

6. 内存分配的技术

内存紧缩（Memory Compaction）

  • 定义：内存紧缩是一种将不连续的空闲内存区域合并为较大空闲区域的技术。通过将所有空闲内存块移动到一起，减少外部碎片的影响，从而提高内存的利用效率。

  • 优点：能够有效减少外部碎片，确保更大的空闲区域供新进程使用。

  • 缺点：

    • 操作过程复杂，需要将进程移动到内存的另一位置。
    • 增加了内存管理的时间开销。
内存管理机制

  • 操作系统通过使用段表和页表来管理内存。段表和页表帮助操作系统跟踪内存中的段和页，确保内存空间的合理分配。

    • 段表：每个进程有一个段表，记录该进程每个段的基址和长度。
    • 页表：进程的虚拟地址通过页表映射到物理地址，以便有效管理内存的分配。

7. 分段与分页管理

分段管理（Segmentation）

  • 定义：分段管理将程序分为多个段，每个段有不同的用途，例如代码段、数据段、堆栈段等。每个段可以独立地进行管理和访问。

  • 优点：

    • 符合程序员的习惯：程序员通常将程序划分为多个段，每个段有不同的用途（如代码、数据、堆栈等）。
    • 便于程序的共享和保护：通过分段，可以在多个进程之间共享某些段，同时保护进程的其他段。

  • 缺点：

    • 外部碎片：由于进程的段大小不一，可能会造成内存的外部碎片。
    • 内存管理复杂：每个段需要独立管理，可能会增加操作系统的管理复杂度。
分页管理（Paging）

  • 定义：分页将内存划分为固定大小的页，每个进程的虚拟地址空间被划分为页。程序通过页表进行映射和地址转换，将虚拟地址映射到物理地址。

  • 优点：

    • 避免外部碎片：通过固定大小的页进行内存分配，避免了内存的外部碎片问题。
    • 内存利用率高：每个进程只使用它所需要的页，内存利用率较高。

  • 缺点：

    • 地址转换开销：进程的虚拟地址需要通过页表转换成物理地址，这增加了地址转换的开销。
    • 不符合程序员直觉：程序员通常按逻辑划分程序的不同部分，但分页管理可能会导致程序的不同部分被分布到不同的物理页上，增加了管理的复杂性。

8. 分段和分页的优缺点

分段的优缺点

  • 优点：

    • 符合程序员的习惯：分段管理与程序员划分程序结构的习惯一致，能够对每个段独立管理。
    • 共享和保护：可以灵活地共享某些段（如代码段），并保护其他段（如数据段、堆栈段）。

  • 缺点：

    • 外部碎片：由于段的大小不一，可能导致内存中的碎片无法被利用。
    • 内存利用率较低：由于进程的段大小不一致，可能导致某些进程无法得到足够的内存资源，造成内存利用率低下。
分页的优缺点

  • 优点：

    • 避免外部碎片：分页管理通过将内存划分为固定大小的页，避免了外部碎片问题，能够提高内存利用率。
    • 内存管理简便：每个进程只需管理其虚拟地址空间的页表，简化了内存管理。

  • 缺点：

    • 增加地址转换的复杂度：分页需要进行虚拟地址到物理地址的转换，增加了系统开销。
    • 不符合程序员的直觉：程序员通常按逻辑分段，但分页可能导致程序的不同部分分布在不同的物理页上，增加了程序的复杂性。

总结

  • 内存分配方案：固定分区和可变分区各有优缺点。固定分区简便但可能浪费内存；可变分区更灵活但可能产生碎片。
  • 内存碎片：内存碎片分为内部碎片和外部碎片，分别由于固定分区和不连续内存分配产生。内存紧缩技术可以减少外部碎片。
  • 内存管理的机制：段表和页表是内存管理的基本机制，用于管理进程的内存地址空间。
  • 分段与分页管理：分段符合程序员的习惯，适用于复杂结构的程序，但会产生外部碎片；分页可以避免外部碎片，但地址转换较复杂。

通过理解这些概念，我们可以更加深入地理解操作系统如何有效地管理内存资源，确保多个进程能够高效共享内存。

9. 分段和分页结合

段页式内存管理结合了分段管理和分页管理的优点，解决了单一分段或分页方法的不足。通过将程序的每个段划分为多个页来进行内存管理，从而达到灵活的内存分配，避免了外部碎片问题。

工作原理：

  1. 程序的划分：

    • 程序被划分为多个段（如代码段、数据段、堆栈段等），每个段可以独立管理，保证程序结构的清晰。
    • 每个段再被进一步划分为多个页，每个页大小固定，解决了分页方式中的外部碎片问题。

  2. 地址转换：

    • 虚拟地址：在段页式管理中，程序的虚拟地址由两个部分组成：

      • 段号：表示程序中的不同段（如代码段、数据段等）。
      • 页号：表示段内的具体页。
      • 页内偏移：表示在页内的偏移量，用于定位具体的数据或指令。

    • 物理地址：系统使用段表和页表进行地址转换。

      • 段表：用于映射段号到物理内存中的段基址。
      • 页表：用于映射页号到物理内存中的页框号。
      • 最终物理地址：通过段表获取段基址，使用页表获取页框号，再加上页内偏移得到最终的物理地址。

  3. 优点：

    • 灵活性：每个段可以根据需要进行动态扩展，并且每个段内的内存分配是基于分页的，因此更灵活。
    • 避免外部碎片：由于分页机制，程序的每个段内部都通过页来管理，避免了传统分段方法中的外部碎片问题。
    • 内存利用率高：结合了分段和分页的优点，既能够合理地管理大程序的不同部分，也能够优化内存的利用率。

  4. 缺点：

    • 复杂性增加：由于结合了分段和分页，系统需要同时管理段表和页表，增加了地址转换的复杂度。
    • 额外的开销：需要维护额外的段表和页表，这会带来一定的系统开销，尤其是当程序的段和页数较多时。

10. 内存管理总结

内存管理的目标：

  • 高效利用内存资源：操作系统的核心任务之一是确保内存资源得到充分的利用。通过合适的内存管理策略，操作系统能够将有限的内存分配给多个进程，确保程序能够顺利执行。
  • 动态增长、共享与保护：内存管理不仅要应对进程动态增长的内存需求，还要保证不同进程之间的内存共享和保护，防止进程间的非法访问。

内存管理的策略：

  1. 连续内存分配：

    • 优点：简单直观，易于实现。
    • 缺点：可能会导致内存碎片，特别是在长时间的运行中。进程之间的内存分配不够灵活。

  2. 分段和分页结合：

    • 优点：结合了分段和分页的优点，能够避免单一方法的缺点。通过灵活的内存分配提高内存利用率，并使得程序的执行更加高效。
    • 缺点：管理复杂，涉及到段表和页表的维护，需要更多的系统资源和时间开销。
    • 总结：段页式内存管理能够有效利用内存，减少碎片，提高系统的内存管理效率，特别适合大型系统或程序。

总结

  • 分段和分页结合：通过将程序的每个段划分为多个页来管理内存，既保持了分段的灵活性，又避免了分页的外部碎片问题，优化了内存利用率。
  • 内存管理的目标是使得多个进程能够共享有限的内存资源，且内存管理不仅仅要确保内存的有效分配，还要保障进程间的内存保护。
  • 内存管理策略包括了固定分区、可变分区、分段、分页和段页式管理等，每种方法都有各自的优缺点，适应不同的需求。段页式管理被认为是最灵活且高效的内存管理策略之一，尤其适用于需要高效内存管理的大型系统。

通过合理的内存管理，可以有效地提升系统性能，减少内存碎片，提高内存利用率。
